EE 194 Berkeley ASIC Tapeout Class — Accelerator Design for VLA Applications

Contributed to accelerator design for VLA (vision-language-action) model. Worked on BF16→FP8 quantization scheme (Max-PO2 per-tensor scaling) for the matrix engines (MXU) and vector unit (VPU) in Pytorch. After RTL-freeze, ran python-based functional models to extract golden data to use in verification and Bring-Up.

Quantization · BF16/FP8 · RTL Modeling · PyTorch · ASIC Tapeout

5-Stage RISC-V CPU with Direct-Mapped Cache, Branch Prediction & Forwarding

Designed a fully functional core with branch prediction and forwarding to minimize stalls, plus a direct-mapped cache to cut memory-access time. Placed 8th among 38 teams on a figure of merit proportional to max frequency and inversely proportional to cycle count and √area — with among the lowest cycle counts.

Verilog · Cache · P&R · VCS · Genus · Innovus · Branch Prediction · Forwarding

Fine-Tuning an LLM to Solve CS 189 MCQs

Fine-tuned Qwen2.5-0.5B-Instruct on Berkeley's CS 189 MCQs using dual-stage mixed fine-tuning to address catastrophic forgetting — Stage 1 on specialized ML MCQs, Stage 2 on a mix of specialized and general MCQs. Reached ~42.5% accuracy on 169 test questions, up from ~28% for the base model.

Supervised Fine-Tuning · Dual-Stage Mixed Fine-Tuning · LLM

Two-Layer Rectilinear Routing Problem

Improved a two-layer routing problem with A* search and a Steiner-tree approximation, reducing total HPWL by 27.6% while running 3× faster.

Routing · A* Algorithm · Steiner Tree · Python

32-bit RISC-V CPU

Built a CPU supporting the 32-bit RISC-V ISA from scratch in Logisim — ALU, control logic, immediate generator, 2-stage pipelining, and data-hazard resolution.

Digital Logic Circuit Design · Computer Architecture

Digit Classification in RISC-V
